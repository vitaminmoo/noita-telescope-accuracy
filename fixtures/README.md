# Memory dumps from a running Noita

These files capture state from a single Noita process so
`scripts/verify.mjs` can run without `noitrainer` (the sibling Go tool
that reads Noita memory) and without a live game.

| File | Source command | Schema |
| --- | --- | --- |
| `biome_flags.ndjson.gz` | `noitrainer-cli biome-flags` | one line per loaded biome chunk: `{cx, cy, name, xmlName, ptr, biomeDataPtr, wobbleEligible, wavyEdge, forceOriginal}`. `name` is the `$biome_*` translation key (shared across sub-variants); `xmlName` is the biome's actual XML filename (the per-chunk identity). |
| `biome_at_resolutions.ndjson.gz` | `noitrainer-cli biome-at-many < coords.txt` | one line per probed world coord: `{wx, wy, gridWidth, gridHeight, xShift, yShift, origCX, origCY, subX, subY, original, resolved, wobbled, wobbleType, neighborDir, neighborCX, neighborCY}` — `original` and `resolved` carry the full BiomeChunkInfo (with `xmlName`). The coords were generated by `scripts/generate.mjs sample-coords --mode=edges` (~84k points: corners + edges + a 16-px sub-grid in the wobble-active strips of every loaded chunk). |
| `pixel_scenes.ndjson.gz` | `noitrainer-cli pixel-scenes` | one line per pixel scene currently queued in the BiomeGrid: `{x, y, materialsFile, colorsFile, backgroundFile, flagSkipBiomeChecks, flagSkipEdgeTextures, index, vec}` (`vec` is `"main"` or `"alt"`). Authoritative answer to "what scenes did the game place?" for diffing against telescope's `loadPixelScene`. |

All three dumps come from the same live Noita session (`scripts/verify.mjs --only=wobble` uses the biome-at fixture; the biomes and pixel-scenes sections use the other two). `_EMPTY_` / `???` chunks are excluded from `biome_flags.ndjson.gz`.

`pixel_scenes_session2.ndjson.gz` is a second pixel-scene dump captured for a different seed, retained as an additional fixture for `scripts/verify.mjs --only=pixel-scenes --scenes=data/dumps/pixel_scenes_session2.ndjson.gz`.

## Seed-1 wobble fixtures

`biome_flags_seed1.ndjson.gz` and `biome_at_seed1.ndjson.gz` are the seed-1 counterparts of the two biome dumps above, captured from a live seed-1 game (matching `data/dumps/full_i`, the entity-harness oracle). The Noita overworld biome **map is seed-independent** (only spawn content is seeded), so the chunk layout matches the 864604507 fixture — these exist to validate the resolver against seed 1 specifically and to back the `mismatch_report.mjs` `⚠wobble` adjudication.

`biome_at_seed1.ndjson.gz` was generated with the new `--mode=dense-diff` (every chunk edge facing a *different-biome* neighbour, sampled at `--step=16` → 109,893 coords). It passes `scripts/verify.mjs --only=wobble --resolutions=data/dumps/biome_at_seed1.ndjson.gz` at `wrong biome: 0`. A full 2-px sweep (6.4M coords, also `wrong: 0`) was run once for confidence but is too large to commit — see `scripts/EDGE_NOISE_VERIFICATION.md` ("Full-density seed-1 wobble validation").

```sh
noitrainer-cli biome-flags | gzip > data/dumps/biome_flags_seed1.ndjson.gz
node scripts/generate.mjs sample-coords --mode=dense-diff --step=16 --input=data/dumps/biome_flags_seed1.ndjson.gz > /tmp/coords.txt
noitrainer-cli biome-at-many < /tmp/coords.txt | gzip > data/dumps/biome_at_seed1.ndjson.gz
```

To regenerate (requires noitrainer + a running Noita):

```sh
noitrainer-cli biome-flags | gzip > data/dumps/biome_flags.ndjson.gz
node scripts/generate.mjs sample-coords --mode=edges --input=data/dumps/biome_flags.ndjson.gz > /tmp/coords.txt
noitrainer-cli biome-at-many < /tmp/coords.txt | gzip > data/dumps/biome_at_resolutions.ndjson.gz
noitrainer-cli pixel-scenes | gzip > data/dumps/pixel_scenes.ndjson.gz
```

`scripts/verify.mjs` reads these dumps directly (`.gz` is decompressed transparently) and compares against noita via `canonicalBiome` in `scripts/_engine_shim.js`, which folds the handful of telescope-vs-XML naming mismatches (tower variants, orbroom variants, temple_altar sub-rooms). To use a different dump, pass `--flags=PATH`, `--resolutions=PATH`, or `--scenes=PATH`.

## Bidirectional dumps

`scripts/dump.mjs` is the telescope-side counterpart — it emits the same
schemas so comparisons can run either direction (telescope-as-source vs.
noita-as-source). Default seed is `786433191` (override via `--seed=N`).

| Subcommand | Telescope | Noita (ground truth) |
| --- | --- | --- |
| `biome-flags` | `node scripts/dump.mjs biome-flags` | `noitrainer-cli biome-flags` |
| `biome-at-many` | `node scripts/dump.mjs biome-at-many < coords.txt` | `noitrainer-cli biome-at-many < coords.txt` |
| `pixel-scenes` | `node scripts/dump.mjs pixel-scenes` | `noitrainer-cli pixel-scenes` |
| `entities` | not yet implemented (browser-only deps on telescope side) | `noitrainer-cli entities-dump` |

Telescope-side caveats:

- `biome-flags` emits all 70×48 biome-map cells (not just "loaded" ones) and
  carries `xmlName` plus the XML-declared edge-noise attrs. It has no
  `wobbleEligible`/`wavyEdge`/`forceOriginal` (those are runtime chunk state).
- `pixel-scenes` covers `STATIC_PIXEL_SCENES` and the `PIXEL_SCENE_BIOMES`
  chunk-color table. PRNG-rolled scenes (friend rooms, watercave, holy
  mountain basins, etc.) are not yet enumerated.
- `entities` is currently a stub — telescope's spawn pipeline transitively
  imports `upng-js` from a CDN URL unusable from Node. Until
  `_engine_shim`-style port of `spawn_functions.js` lands, use
  `noitrainer-cli entities-dump` alone.
